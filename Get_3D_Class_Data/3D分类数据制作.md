# 3D 分类数据制作流程

## 概述

从第一阶段模型输出的 `Prediction_TP_FP_fold_x.csv` 出发，通过灰度区域过滤、置信度分层采样、CT 重采样裁剪，最终生成用于 3D ResNet 二分类训练的标准 patch 数据。(Prediction_TP_FP_fold_x.csv 通过calculate_froc.py来生成)

## 数据来源

| 数据 | 路径 |
|---|---|
| 预测候选框 | `/workspace/data_TP_FP_CSV/Prediction_TP_FP_fold_x.csv` |
| GT 标注 mask | `/data/nifti_files/{mask,collect_test_mask,collect_train_mask,collect_valid_mask}/*.nii.gz` |
| CT 影像 (nifti) | `/data/nifti_files/{image,collect_test_image,collect_train_image,collect_valid_image}/*.nii.gz` |
| Fold 分配 | `/data/yolo_dataset_249/Kfold_neg_0_backup/fold_assignment_integrated.csv` |

## 处理流程

### 步骤 1：提取 GT 3D 检测框

**脚本：** `extract_mask_bboxes.py`

从 3D 粘液栓 mask 文件中提取每个标注区域的 3D 检测框信息。

```
输入:
  - fold_assignment_integrated.csv (fold + image_path)
  - mask .nii.gz 文件 (image_path 中 "image" → "mask")

处理:
  1. 对每个 mask，找到所有非零标签 (1, 2, ..., x)
  2. 对每个标签，计算其 3D bounding box
  3. 通过 affine 矩阵转换到世界坐标 (mm) 和体素坐标

输出:
  - data_TP_FP_CSV/mask_bboxes.csv
    列: patient_key, mask_path, image_path, fold, label_id,
         center_x/y/z (世界坐标), width/height/depth (世界坐标),
         voxel_center_i/j/k (体素坐标), voxel_dim_i/j/k (体素尺寸)
  - 各个粘液栓在其z轴切片的位置
```

**结果：** 184 个 mask → 706 个 GT 检测框

---

### 步骤 2：去除预测中的TP点

**脚本：** `remove_TP_rows.py`

输入: 
  - Get_3D_Class_Data/Prediction_TP_FP_Fold_X.csv

```
处理:
  - 找到表格中所有 表头prediction_type为TP的列，并删除该列

输出:
  - Get_3D_Class_Data/Prediction_FP_Fold_X.csv
```

---
### 步骤 3：区分 FP 置信度

**脚本：** `categorize_confidence.py`

对第一阶段模型输出的 FP 候选，按置信度分档。

```
处理:
  1. 按 userAnnotComment.annotation (模型置信度) 降序排序
  2. 前 60% → 高置信度, 中 30% → 中置信度, 后 10% → 低置信度
  3. 添加 confidence_level 列

原有列: prediction_type = "FP"
```

---

### 步骤 4：合并 GT 到预测表

**脚本：** `merge_gt_to_prediction.py`

将步骤 1 提取的 GT 检测框按 fold 写入对应的 Prediction CSV 文件。

```
处理:
  1. 对 mask_bboxes.csv 中每个 GT box，映射到 Prediction CSV 的列结构
  2. prediction_type = "TP", confidence_level = "GT"
  3. userAnnotComment.annotation = 1.0, iou = 1.0
  4. 按 fold 分别追加到 Prediction_TP_FP_fold_x.csv

合并后 CSV:
  列: StudyInstanceUID, SeriesInstanceUID,
       roi_patientPos_min/max/center_x/y/z, roi_patient_diameter_x/y/z,
       LesionType, userAnnotComment.annotation,
       x, y, z, diameter, diameter_x/y/z,
       path, patient_key, val_fold,
       pixel_*, iou, prediction_type, confidence_level

  prediction_type = "TP"  → GT 样本 (190 个)
  prediction_type = "FP"  → 模型候选 (46,027 个, 已去重)

输出:
- Get_3D_Class_Data/Prediction_GT_FP_Fold_X.csv
```

---

### 步骤 5：提取 3D CT Patch

**脚本：** `extract_patches.py`

这是核心步骤，完成灰度区域过滤、分层采样、CT 重采样和 patch 裁剪。

```
参数:
  --gt_fp_ratio    GT:FP 比例 (默认 0.5 = 1:2)
  --n_jitter       GT 抖动副本数 (默认 10)
  --jitter_range   最大抖动偏移 (默认 4 体素, 约 1.6mm)
  --target_spacing 重采样间距 (默认 0.4, 0.4, 0.8 mm)
  --patch_size     patch 尺寸 (默认 64, 64, 32 体素)
  --hu_min/hu_max  HU 窗 (默认 -1000, 400)
```

#### 5.1 灰度区域过滤

```
FP 中 iou > 0 的行 → 灰度区域，丢弃
FP 中 iou == 0 的行 → 干净负样本，保留
```

106 个 FP 被过滤（全部 IoU < 0.01，与 GT 有微小重叠）。

#### 5.2 分层负样本采样

```
按 confidence_level 从 FP 池中采样:
  高置信度 (60%): 取置信度最高的 n_high 个 — 最难的 hard negative
  中置信度 (30%): 取置信度最高的 n_mid 个
  低置信度 (10%): 取置信度最高的 n_low 个

不足时由剩余池按置信度降序补齐
```

#### 5.3 CT 重采样与裁剪

```
对每个候选 (TP/FP):
  1. 通过 patient_key 定位对应的 nifti CT 文件
  2. 将 bbox 世界中心坐标转换为原始 CT 体素坐标
  3. 重采样 CT 到目标间距 (0.4 × 0.4 × 0.8 mm³)
  4. 计算重采样后的中心坐标
  5. GT: 对中心施加随机抖动 (uniform ± jitter_range 体素)
  6. 裁剪 64 × 64 × 32 体素 patch
  7. HU 窗截断: clip(patch, -1000, 400)
  8. 归一化: (patch + 1000) / 1400 × 255 → uint8
  9. 保存为 .npy 文件
```

#### 5.4 输出结构

```
data_patches/
├── fold_0/
│   ├── 000000_E0001501_TP_j0.npy    # GT 抖动副本 0
│   ├── 000001_E0001501_TP_j1.npy    # GT 抖动副本 1
│   ├── ...
│   ├── 000390_E0001066_FP.npy       # FP 样本
│   ├── ...
│   └── metadata.csv                 # 标注信息
├── fold_1/
│   └── ...
├── fold_2/
├── fold_3/
└── fold_4/

metadata.csv 列:
  fold, patch_idx, patient_key, label (1=TP/0=FP),
  prediction_type, confidence_level, jitter_index,
  center_world_x/y/z, patch_file
```

## 使用方式

```bash
# 步骤 1: 提取 GT bbox
python extract_mask_bboxes.py

# 步骤 2: 去除预测的TP点
python remove_TP_rows.py

# 步骤 3: 给 FP 加置信度标签
python categorize_confidence.py

# 步骤 4: GT 合并到预测表
python merge_gt_to_prediction.py

# 步骤 5: 提取 patch (先 dry_run 验证)
python extract_patches.py --gt_fp_ratio 0.5 --n_jitter 10 --dry_run

# 步骤 5: 正式提取
python extract_patches.py --gt_fp_ratio 0.5 --n_jitter 10
```

## 注意事项

1. **灰度区域判断基于检测级 IoU**（原始 bbox 间），而非 patch 级 IoU（扩展到 64×64×32 后）。检测级 IoU 语义正确，已在 `iou` 列中。
2. **GT 抖动**：每个 GT 生成 10 个独立随机平移后的 patch，避免 bbox 紧贴成为"取景框"捷径，同时增强泛化。
3. **Hard negative 分层**：高置信度 FP（模型最易混淆的负样本）占 60%，确保训练重点放在难分样本上。
4. **HU 窗归一化**：与第一阶段完全一致 (`-1000~400 → 0~255 uint8`)，保证输入分布统一。
5. **坐标转换链**：世界坐标 (mm) → 原始体素 → 重采样体素 → 裁剪 patch，通过 nifti affine 矩阵保证空间一致性。
